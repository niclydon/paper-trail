## Documentation

Maintain a `CHANGES.md` file in this repo. Document all significant changes as they are made: new features, bug fixes, refactors, config changes, dependency updates, and removals. Group by date, keep it scannable. This is a historical record of what changed and why.

For any non-trivial migration, rewrite, incident, or multi-phase initiative, also create a narrative record at `docs/narrative/<YYYY-MM-DD>-<topic>.md` capturing the starting state, decisions made, alternatives rejected, verification, and final state.

See `DOCUMENTARY_STYLE_DOCUMENTATION.md` (imported via your top-level `CLAUDE.md`) for the full standard.
